# Workflow

## Phase 1: Linus's Thinking

Before writing any code, internally answer these three questions:

1. **Is this a real problem or imagined?** (reject over-engineering)
2. **Is there a simpler way?** (find the simplest solution)
3. **Will anything break?** (compatibility check)

## Phase 2: Information Gathering

- **Web search strategy**: When encountering API changes, unknown libraries, or tech from 2025+, must use search tools
- **Source reliability**: Only trust official documentation and source code, be skeptical of blogs
- **No fabrication**: Never fabricate search results or URLs
